🔒

Policy management is restricted to Super Admin. Admin Support does not have the manage-policy permission. Switch to Super Admin to explore the policy editor.

How it works: Adapters (MemoryAdapter + demo-only mutable sources) provide authorization facts. @mizan/core evaluates them against the requested permission, scope, temporal window, and schedule, then returns an explainable allow or deny. Adapters provide facts; Mizan makes the final decision.

⚠ Production security: Hiding or disabling a button in the browser does not enforce authorization. Every protected action in this demo calls the real Mizan decide() before mutating state — the same pattern a production application must repeat on its server or API boundary. The policy editor modifies demo-only in-memory state and is not a real policy administration interface.
