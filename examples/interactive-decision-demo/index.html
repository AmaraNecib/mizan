# Make Model Actions

Demo-only policy controls

These controls modify the in-memory authorization state for Support in real time. Each change triggers a fresh Mizan evaluation — no page reload required.

Current effect:

    Schedule awareness

    A permission can be restricted to a recurring schedule (e.g., business hours). Mizan returns outside-schedule when the evaluation time falls outside the window.

    Admin — reports.read with schedule

    • Monday 10:00 UTC …

    How it works: Adapters (like MemoryAdapter) provide authorization facts — role definitions, assignments, and principal-specific rules. The @mizan/core engine collects those facts, evaluates them against the requested permission, scope, temporal window, and schedule, then returns a final allow or deny with a stable reason code. Adapters provide facts; Mizan makes the final decision.

    ⚠ UI toggles are a demonstration only. Hiding or disabling a button in the browser does not enforce authorization. A production application must repeat every protected check on its server or API boundary using Mizan or an equivalent authorization layer. The policy controls shown here modify only the demo's in-memory state — they are not a real policy administration interface.
