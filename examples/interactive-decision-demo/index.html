# Make Model Actions

How Mizan decides

Adapters provide facts; Mizan makes the final decision. The MemoryAdapter and demo-only mutable sources supply authorization facts — role definitions, assignments, and per-principal overrides. @mizan/core collects those facts, evaluates them against the requested permission, scope, temporal window, and schedule, then returns an explainable allow or deny with a stable reason code.

⚠ Production security: Hiding or disabling a button in the browser does not enforce authorization. Every protected action in this demo calls the real Mizan decide() before mutating state — the same pattern a production application must repeat on its server or API boundary. The policy editor modifies demo-only in-memory state and is not a real policy administration interface.
